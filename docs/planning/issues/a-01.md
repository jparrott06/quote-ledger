## Context

Epic **A — Domain model**: pure, deterministic quote state without I/O. Part of event-sourced quote ledger.

## Requirements

- Internal `QuoteState` models lifecycle before/after quote creation.
- `DomainEvent` models committed facts (initially `QuoteCreated` only).
- `reduce(state, event)` returns the next state or a typed `DomainError`.
- No `seq`, network, or SQLite in this module.

## Acceptance criteria

- [ ] `reduce` rejects a second `QuoteCreated` for the same quote.
- [ ] Public API documented at module level (`src/domain/mod.rs`).
- [ ] CI green.

## Edge cases & failure modes

| Scenario | Expected behavior |
|----------|---------------------|
| `QuoteCreated` when state already has currency | `DomainError::QuoteAlreadyCreated` |
| Replay order preserved | Fold matches `replay()` helper |

## QA / test plan

**Automated**

- Unit tests in `domain` for happy path + duplicate `QuoteCreated`.

**Manual**

- None.

## Rollout / compatibility

Internal API only; proto mapping stays in gRPC layer (later epic).

## Out of scope

- Persistence, `seq`, gRPC mapping, pricing line items.
