## Context

Epic **A — Domain model**: translate **commands** (intent) into **events** (facts) with validation.

## Requirements

- `DomainCommand` models at least `CreateQuote { currency_code, jurisdiction_id }`.
- `command_to_events(state, command)` returns `Vec<DomainEvent>` to append, or `DomainError`.
- Reject `CreateQuote` when a quote already exists.

## Acceptance criteria

- [ ] Blank / whitespace-only `currency_code` or `jurisdiction_id` rejected with `DomainError::InvalidField`.
- [ ] Successful `CreateQuote` emits exactly one `QuoteCreated` event matching fields.
- [ ] CI green.

## Edge cases & failure modes

| Scenario | Expected behavior |
|----------|---------------------|
| Duplicate create | `DuplicateCreateQuote` |
| Leading/trailing spaces in currency | Trim policy: reject if empty after trim OR trim then store — pick one and test |

## QA / test plan

**Automated**

- Unit tests for happy path, duplicate, blank fields.

## Rollout / compatibility

Internal API; gRPC maps proto commands to `DomainCommand` later.

## Out of scope

- Other command types (line items, finalize, …) until Epic E.
