## Context

Epic **A — Domain model**: lock behavior with tests so refactors stay safe.

## Requirements

- Golden-style tests: given an event list, expected final `QuoteState` (or error on invalid stream).
- Property or table-driven tests for `replay` vs manual fold equivalence.

## Acceptance criteria

- [ ] At least one test that exercises `replay` over multiple events (future-proof for more event types).
- [ ] CI green.

## Edge cases & failure modes

| Scenario | Expected behavior |
|----------|---------------------|
| Invalid event mid-stream | `replay` returns `Err` and state is not partially exposed as success |

## QA / test plan

**Automated**

- `cargo test` includes new/expanded domain tests.

**Manual**

- None.

## Rollout / compatibility

Tests only.

## Out of scope

- gRPC integration tests (separate epic).
